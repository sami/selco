Prose Drafts for TMA 02 Sections

3.1 — Project Description: prototype review summary (2–3 sentences)

For use in: Section 1 (Description) / Section 6 (Progress Review)
The first iteration of the project, completed for TMA 01, produced a working prototype comprising three calculator tools — a ten-step Tiling Project Wizard, a Board Coverage Calculator, and a Unit Converter — deployed to GitHub Pages across 105 commits with 100% logic-layer test coverage (Bashraheel, 2026). A structured audit of the prototype against UX evaluation criteria (Tidwell, 2020; Nielsen, 1994) and WCAG 2.1 Level AA identified twenty-two redesign decisions, including a critical phantom token failure that rendered interactive buttons and focus indicators invisible across all wizard and calculator pages. TMA 02 addresses these findings through a consolidated CSS token system, a revised wizard navigation structure, and the addition of masonry calculator modules, building directly on the validated three-layer architecture established in the first iteration.
Citations needed: Bashraheel (2026) for prototype, Tidwell (2020), Nielsen (1994) for evaluation criteria, WCAG 2.1 for accessibility standard.

3.2 — Project Description: problem and solution (2 sentences)

For use in: Section 1 (Description) — showing current state vs. proposed solution concretely
The TMA 01 prototype demonstrated that a mobile-first, multi-material estimation tool is technically feasible using a static Astro architecture with React islands: all three calculator flows produce correct results, the application deploys continuously via GitHub Actions, and the interface renders consistently across desktop, tablet, and mobile viewports (see Figure 7.1–7.8). However, the prototype also revealed that the CSS token system used to implement the Selco brand identity was incomplete, causing key interactive elements — including the wizard's Next button, the Coverage Calculator's Calculate button, and all input focus rings — to render as transparent, creating a functional barrier that must be resolved before any user evaluation can be conducted (see Figures 7.9–7.10).
References Figure 7.1–7.10 from this document. Suitable for a "Current state" sub-section.

3.3 — Work Done section: opening paragraph

For use in: Section 7 (Work Done) — opening paragraph before figures
The TMA 01 prototype was developed across four weekly iterations between 31 January and 3 March 2026 (Appendix 2: Progress Log). The application follows a three-layer architecture — pure calculation logic in TypeScript, project configuration objects, and Astro page routes — a pattern that proved its value when the same tile, adhesive, grout, and spacer functions were reused without modification in both the standalone calculator pages and the ten-step wizard flow (Bashraheel, 2026). Figure 7.1 shows the homepage dashboard as delivered; Figures 7.2 through 7.4 document the Tiling Wizard from initial setup through to the full-project summary; Figures 7.5 and 7.6 show the Board Coverage Calculator and Unit Converter; Figures 7.7 and 7.8 confirm the mobile-responsive layout on a 375 × 812 px viewport. Figures 7.9 and 7.10 annotate the two critical visual bugs identified during the post-iteration UX audit, which TMA 02 addresses as its first priority.
This paragraph introduces all 11 figures and frames TMA01 as a prototype with a specific handover point to TMA02 work.

3.4 — Reflection: one sentence on iteration learning

For use in: Reflection section — following Gibbs' cycle on implementation
The phantom token failure, where twelve CSS custom properties were referenced throughout the application but never defined, illustrates a gap between the intended design system and its implementation: a lesson that informed the TMA 02 decision to add a CI lint step that will fail the build if any undefined token reference is introduced, preventing this category of silent failure from recurring in future iterations.
Applies Gibbs' (1988) cycle: Description → Feelings → Evaluation → Analysis → Conclusion → Action plan. The action plan is the CI lint step (decision T-2 in redesign-decisions.html).
